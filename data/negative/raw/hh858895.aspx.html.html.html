Error ID: 3134

Letzte Aktualisierung: Januar 2013

Betrifft: System Center 2012 - Data Protection Manager, System Center 2012 SP1 - Data Protection Manager

Error ID 3134 — Data Protection Manager

At this time, no additional information is available for this error. The information provided below is identical to the error message content. However, your request for more information has been recorded, and additional information might be added to this topic at a future date.

To provide feedback about the types of information that would help you resolve this error, please contact the DPM Documentation Team.

Details

Product

System Center 2012 – Data Protection Manager (DPM)

ID

3134

Source

Version

4.0

Symbolic Name

Message

DPM could not obtain backup metadata information for <DatasourceType><DatasourceName> on <ServerName>. If the data source is a SharePoint farm, a valid recovery point was created. However, content databases from this recovery point can be recovered only by using the alternate location option.

User Action

If the data source is a SharePoint farm, ensure the following:

  1. That "ConfigureSharePoint.exe -EnabledSharePointProtection" has been run on the front-end Web server with the current SharePoint farm administrator credentials.

  2. That the SharePoint VSS writer is running on the front-end Web server

SQL Server Alias Resolution Failure

Check the job details to see the exact reason for the failure, if it is related to SQL Server alias resolution failure, then check the following:

  • Configure the SQL Server alias on the SharePoint front-end web server to point to the SQL Server or instance of SQL Server of the principal database. Then retry the operation.

  • For SharePoint 2007, ensure that SQL Server Client Connectivity components are installed on the SharePoint front-end web server.

  • Check if you can run "ConfigureSharePoint.exe -ResolveAllSqlAliases" command on the SharePoint front-end web server to identify the SQL Server aliases that cannot be resolved by DPM.
